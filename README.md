## Решение 1: Обеспечение разработки

Решение для обеспечения разработки может быть реализовано при помощи нескольких сервисов:
- Система контроля версий может быть реализована на базе GitHub, GitLab, Bitbucket...
- Репозитории для хранения кода, артифактов, Docker образов могут быть реализованы на базе: Artifactory, Nexus, Harbor... 
- CI-CD инструменты могут быть реализованы при помощи GitLab CI/CD, Jenkins...

Если же такого рода "зоопарк технологий" не приемлем, то со всеми перечисленными требованиями может справиться правильно настроенный GitLab CI/CD:
- Может быть развернут в облаке.
- Содержит в себе систему контроля версий GIT.
- Есть возможность создавать репозитории на каждый сервис.
- Может выступать Docker registry для Docker образов.
- Полноценный функционал CI/CD.
- Безопасность хранения секретов обеспечивается встроенным механизмом CI/CD variables, а также поддержкой HashiCorp Vault.
- Возможность разворачивания агентов сборки на собственных серверах.
- Возможность различных сценариев по запуску тестов и сборок.
